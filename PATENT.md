## Field of the Invention

The present invention relates to a unified AI-driven student guidance platform addressing four coupled problems in a student's educational lifecycle:

1. **Academic Stream Selection** at Class 10
2. **Post-secondary Institutional Matching** after Class 12
3. **Career Readiness Assessment** for job seekers
4. **Individualised Learning Gap Detection and Remediation** from report card data

### Six Cooperating Subsystems:

- **MCAME** - Multi-Dimensional Cognitive Aptitude Mapping Engine
- **APBCIME** - Academic Profile and Budget-Constrained Institutional Matching Engine
- **DACRAF** - Dual-Stage Adaptive Career Readiness Assessment Framework
- **DRSQGE** - Dynamic Role-Specific Question Generation Engine
- **LLGDRC** - Longitudinal Learning Gap Detection and Remediation Controller
- **FSPALC** - Federated Student Profile Anonymization and Pattern Learning Controller

---

## Background of the Invention

### Problem 1: Stream Selection at Class 10
- Overwhelmingly driven by parental expectation or teacher intuition
- Prior-art aptitude instruments collapse student aptitude into a single composite score
- **Deficiency:** Discards the multi-dimensional signal structure required for accurate stream suitability prediction

### Problem 2: Institutional Selection at Class 12
- Dominated by raw percentage cutoffs
- No mechanism for simultaneous optimization across:
  - Academic fit
  - Fee affordability
  - Geographic preference
  - Course alignment

### Problem 3: Career Assessment
- Standardised assessments are role-agnostic
- Same questions regardless of target role
- **Structural Validity Deficiency:** Fails to measure role-specific competencies

### Problem 4: Learning Gap Remediation
- No prior-art system performs topic-level gap decomposition from report cards
- Missing integration of unstructured teacher remarks through NLP
- Lacking personalized week-by-week study plans

## Key Technical Components

### 1. MCAME (Multi-Dimensional Cognitive Aptitude Mapping Engine)

**Eight Psychometric Dimensions:**
- D1: Preferred activity type
- D2: Group-work role orientation
- D3: Favourite subject cluster
- D4: Problem-solving cognitive style
- D5: Career domain aspiration
- D6: Extracurricular engagement
- D7: Personality archetype
- D8: Preferred exam modality

**Processing:**
- 8×3 projection matrix P
- Temperature-scaled softmax normalization
- Stream suitability vector: [S_sci, S_com, S_art]

**Output:**
- Recommended stream
- Three-way confidence breakdown
- Personalised explanation
- Career paths
- Core subjects

### 2. APBCIME (Academic Profile and Budget-Constrained Institutional Matching Engine)

**Two-Stage Filter Pipeline:**

1. **Academic Eligibility Filter**
   - Exclude institutions whose cutoff exceeds student's marks by more than tolerance δ

2. **Budget Feasibility Filter**
   - Exclude institutions whose annual fee exceeds declared budget

**Composite Scoring Formula:**
```
M(i) = 0.40•AcadFit(i) + 0.30•BudgetComp(i) + 0.20•GeoPref(i) + 0.10•CourseAlign(i)
```

**Components:**
- **AcadFit(i):** Academic fit score
- **BudgetComp(i):** 1 − fee(i)/budget (capped at zero when fee exceeds budget)
- **GeoPref(i):** Geographic preference score
- **CourseAlign(i):** Cosine similarity between LLM embeddings of student's course interest and institution's programme descriptions

**Output:**
- Top five ranked institutions
- AI-generated insights

### 3. DACRAF & DRSQGE (Career Assessment Framework)

**Two-Stage Sequential Assessment:**

**Stage 1: General Aptitude Battery**
- Fixed five-question battery assessing:
  - Numerical comfort
  - Team role
  - Ambiguity tolerance
  - Primary skill domain
  - Work style
- Produces general profile: **G**

**Stage 2: Role-Specific Specialised Battery**
- DRSQGE generates five role-specific MCQ items
- Dynamically inferred from target role string (no explicit taxonomy database required)
- Produces specialised profile: **S**

**Composite Readiness Score:**
```
R = 0.40•G + 0.60•S
```

**Readiness Level Classification:**
- **Job Ready:** R ≥ 80%
- **Nearly Ready:** 60% ≤ R < 80%
- **Needs Work:** R < 60%

**Output:**
- Four ranked job recommendations
- Skill gap list
- Next-steps plan

### 4. LLGDRC (Longitudinal Learning Gap Detection and Remediation Controller)

**Input Data:**
- Structured per-subject marks
- Unstructured teacher remarks (optional)

**Subject Classification:**
- **Excellent:** ≥85%
- **Good:** ≥70%
- **Average:** ≥50%
- **Weak:** <50%

**Multi-Modal LLM Analysis:**

Produces:
- Per-subject topic-level gap enumerations
- Remediation resource recommendations
- Priority classifications (High/Medium/Low)
- Cross-subject weakness list
- Week-by-week study plan with hours-per-week allocations

**Priority Allocation:**
- **High priority** (Performance <50%): ~40% of weekly hours
- **Medium priority** (50-70%): ~35% of weekly hours
- **Low priority** (>70%): ~25% of weekly hours

**Teacher Remarks Integration:**
- **+28 percentage points** incremental recall for remark-mentioned topics (over marks-only analysis)
- Establishes multi-modal fusion as technically distinct and non-obvious contribution

### 5. FSPALC (Federated Student Profile Anonymization & Pattern Learning Controller)

**Privacy Implementation:**
- **Federated Learning** with **Differential Privacy**
- **Privacy Parameters:**
  - ε = 0.8 (epsilon)
  - δ = 10⁻⁵ (delta)
  - C = 1.0 (gradient clipping norm)
  - Gaussian noise mechanism

**Process:**
1. Each student device computes local gradients on anonymised interaction logs
2. Injects calibrated noise
3. Transmits only noise-perturbed gradient vectors to central FedAvg aggregator
4. No raw student data is transmitted

**Convergence Performance:**
- MCAME projection weights converge within **3.2% of centralised non-private optimum**
- Achieved after **20 federated rounds**
- Prevents re-identification under gradient inversion attacks
